## What

`synthesizeVerdict` defaults to `claude-haiku-4.5` (cheap/fast) and escalates to `claude-opus-4.7` only when signals are ambiguous or risky.

## Files

- `src/agent/synthesize_verdict.ts` — added `HAIKU_MODEL`, `RISK_KEYWORDS`, `selectSynthesisModel()`, wired into default model selection
- `src/agent/synthesize_verdict_test.ts` — 5 new tests covering routing logic

## Config

No new env vars. `SYNTHESIS_MODEL` still overrides the Opus fallback model (Haiku path is not overridable via env).

## Notes

**Routing logic (`selectSynthesisModel`):**
1. Coverage <50% (resolved < requested/2) → Opus
2. ≤1 category resolved → Opus (confidence would be "low")
3. Risk keywords in findings JSON → Opus (tornado, scam, mixer, tumbler, darknet, phishing, hacker, exploit, rugpull, fraud, malicious, criminal, blocked, ofac-sanctions)
4. Otherwise → Haiku

**Fixture routing on the v8 fixture set:**
- Haiku (6/7 synthesis wallets): Vitalik, Binance, Coinbase, Kraken, Nick, Synthetic fresh wallet
- Opus (1/7): Tornado Cash router (labels contain "tornado-cash")

**Cost math:** Haiku is 5× cheaper per token than Opus ($1/$5 vs $5/$25 per MTok). Routing 6/7 synthesis calls to Haiku → ~68.6% synthesis cost reduction vs the all-Opus v8 baseline.

**Cassette note:** Cassettes match by `METHOD:URL` only (FIFO, body ignored). The synthesis call URL is `POST https://api.agnic.ai/v1/chat/completions` for both models — the cassette serves the prerecorded Opus response regardless of which model the code sends. No re-recording needed.

**Known gap:** The Synthetic fresh wallet routes to Haiku (no risk keywords, coverage ≥2), even though Opus historically returned `confidence: "low"`. In replay this is fine (cassette serves correct response). In production, Haiku may also return `insufficient_data` correctly; if accuracy slips, add low-coverage heuristic or lower the resolved-count threshold.
