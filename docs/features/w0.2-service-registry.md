**What:** Adds a read API + seed script over the curated `service_registry` table so the hot path can pull a vetted x402 service catalog from Postgres instead of re-discovering via Bazaar on every request.

**Files:** src/registry/read.ts, src/registry/types.ts, src/registry/read_test.ts, scripts/seed-registry.ts, db/migrations/0002_service_registry_score.sql, src/db/types.ts

**Config:** DATABASE_URL must be set for reads/seeding; the seeder loads `data/call_recipes.json` (validated via `CallRecipeSchema` from src/discovery/recipe.ts) and upserts on the `resource` URL. Run: `DATABASE_URL=<neon-url> ~/.deno/bin/deno run --allow-net --allow-env --allow-read scripts/seed-registry.ts`.

**Notes:** `service_registry` table was already created in 0001_init.sql (W0.1); migration 0002 only adds `score numeric(5,4) DEFAULT 1.0` and `last_vetted_at timestamptz`. `read.ts` exposes `getActiveServices(category)` (status='active' only) and `getRecipe(service_id)`. All seeded entries start status='active', score=1.0, last_vetted_at=now(). Consumed by W0.4 (hot-path discovery swap) and W0.9 (score-based ranker).
