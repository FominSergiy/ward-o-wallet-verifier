# w0.5-per-call-timeouts-caching

**What:** Bounds tail latency from x402 service calls with a per-call timeout (default 2s), and caches ENS reverse-resolution and eth-labels lookups for 24h to avoid redundant network calls.

**Files:**
- `src/cache/kv.ts` — new KvStore abstraction (MemoryKv default, Deno KV when `CACHE_BACKEND=kv`); exports `getKv()`, `newMemoryKv()`, `_overrideKvForTests()`
- `src/agent/invoke_all.ts` — `withInvokeTimeout()` wraps every `invoker()` call in `Promise.race` against a configurable timeout; timeout resolves to an error outcome, triggering normal alternate fallback logic. Timer is cleared via `finally` to avoid Deno test runner leak warnings.
- `src/agent/ens_resolver.ts` — `resolveEns()` checks cache before RPC call; stores result keyed by `ens:<address_lc>:<chain>`
- `src/agent/labels_registry.ts` — `fetchLabelsRegistry()` checks cache before HTTP fetch; stores result keyed by `eth_labels:<address_lc>`
- `src/agent/invoke_all_test.ts` — new test: slow service (never resolves) times out and is counted as error
- `src/agent/ens_resolver_test.ts` — new test: second ENS call for same address returns cached result (getEnsName called once); existing tests updated with `cache: newMemoryKv()` for isolation
- `src/agent/labels_registry_test.ts` — existing tests updated with `cache: newMemoryKv()` for isolation; new cache-hit test added

**Config:**
- `INVOKE_TIMEOUT_MS` — per-call timeout in ms (default: `2000`)
- `ENS_CACHE_TTL_MS` — ENS cache TTL in ms (default: `86400000` = 24h)
- `ETH_LABELS_CACHE_TTL_MS` — eth-labels cache TTL in ms (default: `86400000` = 24h)
- `CACHE_BACKEND` — `"kv"` for Deno KV persistence, anything else for in-process MemoryKv

**Notes:**
- The non-eth chain path in `resolveEns` returns early before the cache check (chain not supported → result is deterministic, not worth caching).
- The `finally(() => clearTimeout(timerId))` pattern in `withInvokeTimeout` is required to prevent leaked timer warnings in Deno's test runner when the invoker resolves before the timeout fires.
- Tests that share wallet addresses must pass `cache: newMemoryKv()` per-call; the module-level singleton persists across tests in a single run.
