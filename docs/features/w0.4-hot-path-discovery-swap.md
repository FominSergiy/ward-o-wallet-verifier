# W0.4 — Hot-path discovery swap

**What:** The `/verify-agent` request path now builds its service plan by reading the curated `service_registry` instead of fanning out to Bazaar (CDP x402 discovery) + LLM rerank on every request. Discovery becomes a background/maintenance concern; the hot path is a deterministic registry read.

## Files

- `src/registry/select.ts` (new) — `selectFromRegistry(address, categories, opts)` → `DiscoveryPlan`. Takes the top-scored active registry entry per category as the primary service and the rest as runtime alternates; reconstructs each service's call shape (`inputInfo`) from its recipe. Offline-safe.
- `src/registry/read.ts` — added `loadAllRecipes()` (validates every recipe in `data/call_recipes.json`, keyed by `service_id`); `getRecipe()` now delegates to it.
- `src/registry/select_test.ts` (new) — 6 unit tests: offline fallback, score-descending ranking, unresolved categories, missing-recipe skip, call-shape reconstruction, zero `searchDiscovery`.
- `src/agent/verify.ts` — hot path swapped from `discover()` → `selectFromRegistry()`. `_testHooks.discover` renamed to `_testHooks.selectFromRegistry`. Emitted phase keeps the `discover` label for SSE/UI contract stability.
- `src/agent/verify_test.ts`, `src/agent/events_test.ts` — test seam renamed to match.
- `src/routes/verify_agent.ts` — added `verify` injection seam to `VerifyAgentRouterOpts` (defaults to real `verifyAgent`).
- `src/routes/verify_agent_test.ts` — hermetic budget-guard tests now inject a fast-failing `verify` stub (the registry path no longer fails fast on a missing API key, so without this the pipeline made real oracle/x402 calls that leaked under the sanitizer).
- `scripts/validate_w04.ts` (new) — one-shot paid validation pass: times the pipeline, records fresh cassettes, and diffs the select phase against `benchmarks/v8-baseline.json`.
- `tests/cassettes/*.json` — re-recorded against the registry path.

## Config

None added. Uses the existing `DATABASE_URL` (registry reads) and `data/call_recipes.json` (call shapes). No new env vars.

## Notes / gotchas

- **Network detection dropped from the hot path.** The old `discover()` called `detectWalletNetwork()` (Agnic `/api/balance`). Recipes are pinned to Base mainnet, so `selectFromRegistry` defaults `walletNetwork` to `base` with no balance call. This is also why the route's hermetic tests needed the `verify` stub — they previously relied on `discover()` throwing on the missing `AGNIC_API_KEY`.
- **Live DB is missing migration 0002.** The Neon branch in `.env` has `service_registry` without the `score` column, so `getActiveServices()` throws and `selectFromRegistry` falls back to `call_recipes.json` (score 1.0). The DB-backed ranking is not actually live until `deno task db:migrate` + `scripts/seed-registry.ts` run against that branch. Follow-up.
- **`web_sentiment` recipe price drift.** `crypto-news-sentiment-api-628b81` now demands more than its frozen `price_usdc` (0.005) → `Payment exceeds maximum allowed value` on every call. Verdicts are unaffected (web_sentiment is supplementary), but the recipe should be re-snapshotted. Follow-up.
- **Ranking is still static.** Every active row is `score = 1.0`; nothing recomputes scores yet. The feedback loop — W0.9 (score-based ranker, reads `service_observations`/`service_health_durable`) and W0.10 (background vetter that re-runs `discover()` off the hot path to admit/demote services) — is the maintenance side that keeps this registry fresh, and is not built yet.
