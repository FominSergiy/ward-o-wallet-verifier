# w0.7-verdict-cache

**What:** KV-backed verdict cache that short-circuits `verifyAgent` before any oracle or service calls when the same `(chain, address, schema_version)` tuple has been seen recently.

**Files:**
- `src/agent/verdict_cache.ts` (new) — `VerdictCache` interface, `denoKvCache(kv)` factory, `memoryCache()` for tests, TTL constants
- `src/agent/verdict_cache_test.ts` (new) — unit tests for `memoryCache` + `verifyAgent` cache integration
- `src/agent/verify.ts` — imports `VerdictCache`, adds `verdictCache` opt to `VerifyAgentOpts`, checks cache before oracle, writes cache on both exit paths

**Config:** No new env vars. Callers opt in by passing `verdictCache: denoKvCache(await Deno.openKv())` to `verifyAgent`. Omitting it preserves existing no-cache behavior.

**Notes:**
- TTLs: 24h for `safe_to_transact`, 5min for `do_not_transact` (short to limit false-positive blast radius), no cache for `insufficient_data`.
- Cache key includes `SCHEMA_VERSION = "1"` — bump this constant when `WalletVerdict` shape changes to avoid serving stale entries with wrong structure.
- The oracle short-circuit (sanctioned address) also writes to cache so a repeat check is instant and free.
- The route layer (`src/routes/verify_agent.ts`, `src/mcp/`) needs to construct and pass a `denoKvCache` instance — that wiring is left for the caller (W0.7 only defines the mechanism).
