# W1.5 — WARD-o x402 self-listing + Bazaar metadata

Part of the [pivot plan](README.md). Workstream 1. Rail B prep — WARD-o publishes itself as a Bazaar service so x402-capable agents can discover and pay it directly. Depends on W0.10 because synthesis cost must be stable before we quote a price.

**Workstream:** 1
**Dependencies:** W0.10 (cost telemetry stable), W1.2 (usage_events populated)
**Status:** planned

## Acceptance criteria

- WARD-o has a Bazaar listing under `wallet_verification` category with: resource URL, description, accepts (USDC on base), price (initial value can be hand-set), input schema (`{address, chain, budgetCeiling?}`), expected output (`WalletVerdict`).
- Listing is registered via the appropriate Bazaar publish flow (document the steps).

## Files

- `scripts/publish-bazaar-listing.ts` (new)
- `docs/x402-listing.md` (new)

## Validation

- Manual: confirm listing appears in Bazaar search results for the category.

## Test spec

- Smoke test: a separate x402-capable client can discover WARD-o via `searchDiscovery()` and parse the listing.
