# W1.3 — Stripe billing integration (MCP rail)

Part of the [pivot plan](README.md). Workstream 1. Rail A — persistent integrations (agent platforms, dev tools) bill via Stripe + API key.

**Workstream:** 1
**Dependencies:** W1.1, W1.2
**Status:** planned

## Acceptance criteria

- Stripe customer per tenant. Usage-based price configured.
- Daily roll-up job: sum `usage_events.total_cost_usd` per tenant per day, send Stripe usage record.
- Webhook handler: on past-due, mark tenant `over_quota` and reject requests with 402.
- Free tier (configurable, default 100 checks/mo) before usage records start incrementing.

## Files

- `src/billing/stripe.ts` (new)
- `src/billing/rollup.ts` (new) — daily job
- `src/routes/stripe_webhook.ts` (new)
- `scripts/rollup-usage.ts` (new) — manual trigger

## Validation

- `~/.deno/bin/deno check src/billing/*.ts`
- Manual: run rollup with test data, verify Stripe dashboard shows usage.

## Test spec

- Test: rollup sums correctly and skips already-rolled days.
- Test: webhook flips tenant status to `over_quota`.
- Test: `over_quota` tenant hitting verify gets 402.
