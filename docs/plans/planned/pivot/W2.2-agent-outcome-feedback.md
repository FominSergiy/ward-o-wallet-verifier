# W2.2 — Agent outcome feedback endpoint

Part of the [pivot plan](README.md). Workstream 2. Closes the loop the moat depends on. Same `agent_outcomes` data later feeds W4 graph-model labels.

**Workstream:** 2
**Dependencies:** W1.1 (need tenant context)
**Status:** planned

## Acceptance criteria

- `POST /report-outcome` accepts `{verification_id, outcome: "settled"|"reverted"|"abandoned"|"disputed", notes?}`.
- Authenticated via same bearer flow as other routes.
- Writes to a new `agent_outcomes` table linked to `usage_events` by `verification_id` (= request_id).

## Files

- `src/routes/report_outcome.ts` (new)
- `db/migrations/000X_*.sql` — `agent_outcomes` table
- `src/main.ts` — route mount

## Validation

- `~/.deno/bin/deno check src/routes/report_outcome.ts`
- `~/.deno/bin/deno task test:unit`

## Test spec

- Test: valid outcome report → 200, row written.
- Test: unknown `verification_id` → 404.
