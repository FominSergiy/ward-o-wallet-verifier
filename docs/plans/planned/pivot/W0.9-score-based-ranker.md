# W0.9 — Score-based ranker

Part of the [pivot plan](README.md). Workstream 0. Deterministic, no-LLM ranking driven by observed reliability. Replaces the per-request LLM rerank.

**Workstream:** 0
**Dependencies:** W0.2, W0.8
**Status:** planned

## Acceptance criteria

- `src/registry/score.ts` exposes `recomputeScores()` that reads the trailing-30-day `service_observations`, computes per-service score from reliability, P95 latency, coverage rate, updates `service_registry.score` and `status`.
- Status transitions implemented: `probation → active` after N successful windows; `active → probation` after sustained drop; `probation → blocked` after persistent failure; `blocked → probation` only via explicit retry.
- `selectFromRegistry()` (W0.4) consumes `score` and `status` deterministically — no LLM in the ranker.

## Files

- `src/registry/score.ts` (new)
- `src/registry/select.ts` — update to honor score + status

## Validation
