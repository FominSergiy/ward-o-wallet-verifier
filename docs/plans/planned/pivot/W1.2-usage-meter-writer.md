# W1.2 — Usage meter writer

Part of the [pivot plan](README.md). Workstream 1. Records every paid call for both Stripe roll-up (Rail A) and x402 cost-based pricing (Rail B). Single schema feeds both rails.

**Workstream:** 1
**Dependencies:** W0.1, W1.1, W−1.1
**Status:** planned

## Acceptance criteria

- Every verify request writes to `usage_events`: `{tenant_id, request_id, route, wallet_address, chain, verdict, total_cost_usd, llm_cost_usd, x402_cost_usd, latency_ms, timestamp}`.
- Write is async, non-blocking.
- An anonymous tenant id (`x402_anonymous`) is used when there is no API key (placeholder until W1.6 wires real x402).

## Files

- `src/usage/meter.ts` (new)
- `src/routes/verify_agent.ts`, `src/routes/verify_agent_stream.ts`, `src/mcp/server.ts` — call meter

## Validation

- `~/.deno/bin/deno check src/usage/meter.ts`
- `~/.deno/bin/deno task test:unit`

## Test spec

- Test: one verify call produces one `usage_events` row with correct fields.
- Test: meter failure does not break verify response.
