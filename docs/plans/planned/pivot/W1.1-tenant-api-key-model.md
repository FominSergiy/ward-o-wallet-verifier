# W1.1 — Tenant + API key model

Part of the [pivot plan](README.md). Workstream 1 (Sellable wedge). Replaces the single `MCP_SHARED_SECRET` with per-tenant keys across both HTTP and MCP transports.

**Workstream:** 1
**Dependencies:** W0.1
**Status:** planned

## Acceptance criteria

- `tenants` and `api_keys` tables populated. API keys stored as `sha256` hashes, never plaintext.
- New `src/auth/api_key.ts` validates `Authorization: Bearer <key>` → tenant or returns 401.
- All HTTP routes (`src/routes/`) and MCP Streamable HTTP transport (`src/mcp/http.ts`) gated by `requireTenant()` middleware.
- `MCP_SHARED_SECRET` is retired. README updated.

## Files

- `src/auth/api_key.ts` (new)
- `src/main.ts` — wire middleware
- `src/mcp/http.ts` — swap auth
- All `src/routes/*.ts` — apply middleware

## Validation

- `~/.deno/bin/deno check src/auth/api_key.ts src/main.ts`
- `~/.deno/bin/deno task test:unit`

## Test spec

- Test: request without bearer → 401.
- Test: request with valid bearer → resolves tenant, attaches to context.
- Test: invalid bearer → 401.
