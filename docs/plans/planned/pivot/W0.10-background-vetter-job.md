# W0.10 — Background vetter job (cron)

Part of the [pivot plan](README.md). Workstream 0. Moves the existing `discover()` pipeline (LLM rerank + param-shape discovery) out of the hot path and into a scheduled maintenance job.

**Workstream:** 0
**Dependencies:** W0.2, W0.3, W0.9
**Status:** planned

## Acceptance criteria

- A cron job (Cloudflare Cron Triggers or GitHub Actions, choose during ticket execution) runs once per 12h.
- Job runs the existing `discover()` pipeline for each category. New candidates → `probation`. Calls `recomputeScores()` (W0.9). Promotes/demotes per status machine.
- LLM-based param-shape discovery runs **only** here, not in the hot path.
- Manual trigger: `scripts/vet.ts` runs the same job once on demand.

## Files

- `scripts/vet.ts` (new) — same code as the cron handler
- `src/vetter/run.ts` (new) — main logic
- Deployment config (Wrangler / GHA YAML) — committed

## Validation

- `~/.deno/bin/deno check src/vetter/run.ts scripts/vet.ts`
- Manual: run `scripts/vet.ts`, verify registry changes are sane.

## Test spec

- Test: vetter finds a new candidate, inserts as `probation` with `last_vetted_at` set.
- Test: vetter respects `blocked` status — doesn't re-attempt.
