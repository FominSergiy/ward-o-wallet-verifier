# W1.6 — x402 payment receiver on `/verify-agent`

Part of the [pivot plan](README.md). Workstream 1. Rail B — anonymous, pay-per-call. The payment is the auth.

**Workstream:** 1
**Dependencies:** W1.5
**Status:** planned

## Acceptance criteria

- `/verify-agent` accepts an x402 `Payment-Required` flow: first request returns 402 with payment instructions; second request includes `X-PAYMENT` header with signed USDC payment.
- Payment is verified server-side via the appropriate x402 facilitator.
- On success, `usage_events.tenant_id = "x402_anonymous"` row written with `x402_payment_id`.
- No API key required when payment is present.

## Files

- `src/auth/x402.ts` (new) — payment verification
- `src/routes/verify_agent.ts` — accept either bearer or x402 payment

## Validation

- `~/.deno/bin/deno check src/auth/x402.ts`
- E2E: a test x402 client successfully pays and gets a verdict.

## Test spec

- Test: request with no auth and no payment → 402 with payment instructions.
- Test: request with valid payment → 200 with verdict.
- Test: request with replayed payment → 402.
