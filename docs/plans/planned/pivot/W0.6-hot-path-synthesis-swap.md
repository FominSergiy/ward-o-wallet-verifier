# W0.6 — Hot-path synthesis swap: Haiku/Sonnet default, Opus on ambiguity

Part of the [pivot plan](README.md). Workstream 0. Largest single latency + cost lever. Opus stays as a fallback for ambiguous wallets and as the background re-evaluation model.

**Workstream:** 0
**Dependencies:** W0.4, W−1.3 (cassettes for regression)
**Status:** planned

## Acceptance criteria

- `src/agent/synthesize_verdict.ts` defaults to `claude-haiku-4-5` (or `claude-sonnet-4-6` if accuracy slips).
- Triggers Opus fallback when: (a) findings conflict, (b) coverage <50%, (c) confidence would be `low` per current logic.
- `test:replay` accuracy ≥ 8/9; if 9/9 only with Opus fallback active, that's acceptable but documented.
- Per-call synthesis cost drops by ≥40% on the fixture set average (vs. v8 baseline in `benchmarks/v8-baseline.json`).

## Files

- `src/agent/synthesize_verdict.ts` — model routing
- Prompt tuning may live in this file or `src/agent/prompts/`

## Validation

- `~/.deno/bin/deno check src/agent/synthesize_verdict.ts`
- `~/.deno/bin/deno task test:replay` — accuracy gate.
- `~/.deno/bin/deno task benchmark` — compare to v8 baseline; record new numbers.

## Test spec

- Test: ambiguous fixture wallet routes to Opus path.
- Test: clean-and-clean fixture wallet uses Haiku and gets correct verdict.
