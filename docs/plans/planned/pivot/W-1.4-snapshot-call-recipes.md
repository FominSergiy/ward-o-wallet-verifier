# W−1.4 — Snapshot current call recipes to `data/call_recipes.json`

Part of the [pivot plan](README.md). Workstream −1 (Telemetry baseline). Captures implicit knowledge in today's pattern-adapter / descriptor-retry logic so W0 can move param discovery out of the hot path without losing it.

**Workstream:** −1
**Dependencies:** W−1.1
**Status:** planned

## Acceptance criteria

- New `scripts/snapshot-recipes.ts` runs the existing `discover()` pipeline once per category, captures each successful service's `{resource, method, path_params, query_params, body_schema, body_type, pay_to, network, price}` and writes `data/call_recipes.json` keyed by `service_id` (hash of `host + payTo + network`).
- The snapshot includes at least one working service per category for: sanctions, labels, onchain_history, web_sentiment.
- File is committed.

## Files

- `scripts/snapshot-recipes.ts` (new)
- `data/call_recipes.json` (new, committed)
- `src/discovery/recipe.ts` (new helper for the recipe shape)

## Validation

- `~/.deno/bin/deno check scripts/snapshot-recipes.ts src/discovery/recipe.ts`
- `~/.deno/bin/deno run --allow-net --allow-env --allow-write scripts/snapshot-recipes.ts` — produces the file.

## Test spec

- Smoke test: load `data/call_recipes.json` and assert it has ≥1 entry per category.
- Schema test: every entry parses as the `CallRecipe` type.
