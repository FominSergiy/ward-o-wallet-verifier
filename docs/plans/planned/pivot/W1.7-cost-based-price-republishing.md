# W1.7 — Cost-based price republishing job

Part of the [pivot plan](README.md). Workstream 1. Makes the x402 quoted price track WARD-o's own observed cost basis, so margin doesn't silently collapse.

**Workstream:** 1
**Dependencies:** W1.2, W1.5
**Status:** planned

## Acceptance criteria

- Weekly job computes rolling 30-day P50 of `usage_events.total_cost_usd`.
- Multiplies by markup (default 3×, configurable) to produce new quoted price.
- Updates Bazaar listing price. Logs the change.

## Files

- `scripts/republish-price.ts` (new)
- `src/pricing/compute.ts` (new)

## Validation

- `~/.deno/bin/deno check src/pricing/compute.ts`
- Manual: run job, verify Bazaar listing price updated.

## Test spec

- Test: given a fixture set of `usage_events` rows, compute returns expected P50 × markup.
