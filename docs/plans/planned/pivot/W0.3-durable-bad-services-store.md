# W0.3 — Durable bad-services store (migrate from health_store.json)

Part of the [pivot plan](README.md). Workstream 0. Fixes the "we keep re-discovering broken services" bug — health data was in-memory on Deno Deploy and reset on cold start.

**Workstream:** 0
**Dependencies:** W0.1
**Status:** planned

## Acceptance criteria

- `src/discovery/health_store.ts` writes to `service_health_durable` table instead of JSON file / in-memory Map.
- Existing durable-block codes still trigger sticky blocks (`unsubstituted_path_param`, `not_found`, `non_json_response`, `payment_exceeds_max`).
- On Deno Deploy cold start, blocks persist (was the bug).
- Backwards-compatible read API; callers don't change.

- `db/migrations/0002_*.sql` — `service_health_durable` indexes
