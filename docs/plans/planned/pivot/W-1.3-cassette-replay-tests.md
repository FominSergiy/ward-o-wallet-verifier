# W−1.3 — Cassette-based replay test mode

Part of the [pivot plan](README.md). Workstream −1 (Telemetry baseline). Enables cheap regression checks for W0 iteration.

**Workstream:** −1
**Dependencies:** W−1.1
**Status:** planned

## Acceptance criteria

- Recording mode: a one-shot script captures real x402/HTTP responses for each fixture wallet into `tests/cassettes/<wallet>.json` (request URL + body → response body, status, headers).
- Replay mode: a fetch interceptor reads from cassettes; missing cassette entries fail loudly.
- New `deno task test:replay` runs the verification logic against the fixture suite in replay mode in <10s with zero paid calls.
- Replay test asserts 9/9 verdict accuracy.

## Files

- `tests/cassettes/` (new dir)
- `scripts/record-cassettes.ts` (new)
- `tests/replay_test.ts` (new)
- `src/testing/fetch_interceptor.ts` (new helper)
- `deno.json` — `test:replay` task

## Validation

- `~/.deno/bin/deno check scripts/record-cassettes.ts tests/replay_test.ts`
- `~/.deno/bin/deno task test:replay` — passes offline with `--allow-read` only.

## Test spec

- `tests/replay_test.ts` — for each fixture wallet, run verify, assert verdict equals expected, assert no real network call was made.
