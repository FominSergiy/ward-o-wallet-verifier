# W2.1 — Extend `service_observations` for severity contribution

Part of the [pivot plan](README.md). Workstream 2 (Reputation instrumentation). Adds the columns needed to differentiate filler signals from verdict-moving ones.

**Workstream:** 2
**Dependencies:** W0.8
**Status:** planned

## Acceptance criteria

- `service_observations` gains: `tenant_id`, `finding_severity` (the per-service finding severity), `finding_contributed_to_verdict` (bool, did this finding actually move the synthesis verdict).
- W0.8's writer is updated to populate these from the synthesis trace.

## Files

- `db/migrations/000X_*.sql`
- `src/observability/observations.ts` — extended writer
- `src/agent/synthesize_verdict.ts` — emit per-service contribution info

## Validation

- `~/.deno/bin/deno check src/observability/observations.ts src/agent/synthesize_verdict.ts`

## Test spec

- Test: a verify call where a labels service returns "exchange" produces an observation row with `finding_contributed_to_verdict = true` if that finding was in the final verdict.
