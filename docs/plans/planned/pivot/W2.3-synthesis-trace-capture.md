# W2.3 — Synthesis trace capture

Part of the [pivot plan](README.md). Workstream 2. Lets W2.1 know which findings actually drove the verdict and lets W2.4 score services by verdict contribution.

**Workstream:** 2
**Dependencies:** W0.6
**Status:** planned

## Acceptance criteria

- `synthesizeVerdict()` returns, alongside the verdict, a `trace: { signal_id, used_in_verdict: bool, weight }[]`.
- Trace is persisted to `synthesis_traces` table or as a JSONB column on `usage_events`.

## Files

- `src/agent/synthesize_verdict.ts`
- `db/migrations/000X_*.sql`

## Validation

- `~/.deno/bin/deno check src/agent/synthesize_verdict.ts`
- `~/.deno/bin/deno task test:replay`

## Test spec

- Test: trace contains an entry for every input finding.
- Test: at least one entry has `used_in_verdict = true` when a finding is referenced in `reasoning`.
