# W0.5 — Per-call timeouts + ENS/eth-labels caching

Part of the [pivot plan](README.md). Workstream 0. Bound the tail latency contribution from each downstream service; cache auxiliary lookups.

**Workstream:** 0
**Dependencies:** W0.4
**Status:** planned

## Acceptance criteria

- Every x402 service invocation in `invokeAll()` has a 2s per-call timeout. Timeout → counted as error, fall back to next-ranked service.
- ENS reverse-resolution result cached by `(address, chain)` for 24h.
- eth-labels.com lookup cached by `address` for 24h.
- Cache TTLs configurable via env.

## Files

- `src/agent/invoke_all.ts` — timeout wrapping
- `src/agent/ens.ts` (if exists; else inline) — cache
- `src/agent/eth_labels.ts` — cache
- KV setup: `src/cache/kv.ts` (new, picks Deno KV vs. memory based on env)

## Validation

- `~/.deno/bin/deno check src/agent/invoke_all.ts src/cache/kv.ts`
- `~/.deno/bin/deno task test:unit`

## Test spec

- Test: a service that delays 3s is treated as error.
- Test: second call to ENS for same address hits cache (no network).
