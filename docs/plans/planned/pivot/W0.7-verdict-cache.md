# W0.7 — Verdict cache in KV

Part of the [pivot plan](README.md). Workstream 0. Sub-second responses on repeated wallet checks; major margin lever for the paid wedge.

**Workstream:** 0
**Dependencies:** W0.4
**Status:** planned

## Acceptance criteria

- Verdict cache keyed by `(chain, address, schema_version)`.
- TTL: 24h for `safe_to_transact`, 5min for `do_not_transact` (insurance against false positives), no cache for `insufficient_data`.
- Cache check happens before any service calls. Cache hit returns in <100ms.

## Files

- `src/agent/verdict_cache.ts` (new)
- `src/agent/verify.ts` — short-circuit on hit

## Validation

- `~/.deno/bin/deno check src/agent/verdict_cache.ts`
- `~/.deno/bin/deno task test:unit`

## Test spec

- Test: second call for same address returns cached verdict, no service calls made.
- Test: `do_not_transact` verdict TTL is 5min not 24h.
