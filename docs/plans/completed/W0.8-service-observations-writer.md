# W0.8 — `service_observations` event log writer

Part of the [pivot plan](README.md). Workstream 0. Same writer also feeds Workstream 2 reputation scoring and W4 graph-training labels.

**Workstream:** 0
**Dependencies:** W0.1, W−1.1
**Status:** completed

## Acceptance criteria

- Every `ServiceEvent` is also written to `service_observations` table with columns: `request_id`, `service_id`, `service_resource`, `category`, `network`, `price_usdc`, `latency_ms`, `status`, `error_code`, `response_shape_match`, `timestamp`.
- Writes are async / fire-and-forget (don't slow the request).
- Failure to write logs an error but doesn't break the verify flow.

## Files

- `src/observability/observations.ts` (new)
- Hook into wherever `ServiceEvent` is emitted

## Validation

- `~/.deno/bin/deno check src/observability/observations.ts`
- `~/.deno/bin/deno task test:unit`

## Test spec

- Test: a verify call produces N `service_observations` rows where N = number of x402 services called.
- Test: a DB write failure does not throw out of the verify call.
