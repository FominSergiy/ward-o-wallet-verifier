# W0.4 — Hot-path discovery swap: registry instead of Bazaar+LLM rerank

Part of the [pivot plan](README.md). Workstream 0. The architectural flip — discovery becomes a background maintenance job; the request path just reads the registry.

**Workstream:** 0
**Dependencies:** W0.2, W0.3
**Status:** completed

## Acceptance criteria

- `verify()` no longer calls `discover()` in the request path; instead calls `selectFromRegistry(categories)`.
- `discover()` is preserved but only invoked by the background vetter (W0.10).
- A replay-mode verify on fixtures hits zero Bazaar API calls and produces the same verdict.

## Files

- `src/agent/verify.ts` — swap discovery call
- `src/registry/select.ts` (new) — picks top-N active services per category

## Validation

- `~/.deno/bin/deno check src/agent/verify.ts src/registry/select.ts`
- `~/.deno/bin/deno task test:replay` — passes, no Bazaar calls.

## Test spec

- Test: fixture run produces 9/9 verdicts using only registry-selected services.
- Test: zero invocations of `searchDiscovery()` during a verify call.
